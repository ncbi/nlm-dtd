<glossary>

Glossary Elements List

A section whose content is typically a list of terms and definitions, usually tagged as one or more two-part lists called Definition Lists <def-list>s. Typically part of the back matter of an article.

Remarks

The model for <glossary> is broad in order to allow examples found in literature that are tabular, heavily annotated, constructed as a series of notes, or otherwise unusually constructed to be captured in this Tag Suite.

Glossaries are recursive with various levels indicated by containment, not by different names for the embedded glossaries. A typical simple <glossary> might contain only a title and a <def-list>. A more complex <glossary> might be divided into several different list of terms, each headed by a title that described the audience for the particular terms. These glossaries inside a glossary are tagged using the <glossary> element rather than a different, explicitly named element such as a <sub-glossary> or <glossary-sec>.

Conversion Note: Versions 1 and 2 of this Tag Suite modeled this recursive structure with a differently named interior list (<gloss-group>). That element can be directly translated to a <glossary>.

This element (<glossary>) has been significantly remodeled in version 3.0 of the Tag Set. The values and/or usage in version 3.0 are not backward-compatible with that in previous versions of the Tag Set. Specifically, in prior versions some of the function of the <glossary> element was handled via the <gloss-group> element, which no longer exists.

Attributes

content-type Type of Content
id Identifier
specific-use Specific Use

Content Model

<!ELEMENT  glossary     %glossary-model;                             >

Expanded Content Model

(title?, p*, def-list*, glossary*)

Description

The following, in order:

  • <title> Title, zero or one
  • <p> Paragraph, zero or more
  • <def-list> Definition List, zero or more
  • <glossary> Glossary Elements List, zero or more

This element may be contained in:

<back>, <glossary>
